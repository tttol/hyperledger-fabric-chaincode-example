# ピアノードとは？
Hyperledger Fabricにおける**ピアノード（Peer Node）**は、ネットワーク上で重要な役割を果たすノードの一つです。ピアノードは以下のような機能や役割を担っています。

1. トランザクションの処理と検証

ピアノードは、スマートコントラクト（Hyperledger Fabricでは「チェーンコード」と呼ばれます）を実行し、トランザクションを処理します。具体的には、クライアントアプリケーションから送られてくるトランザクションリクエストを受け取り、その内容を実行・検証します。

2. トランザクションのコミット

ピアノードは、トランザクションが検証され、コンセンサスが得られた後、ブロックチェーンにトランザクションをコミットします。ピアノードがトランザクションを承認した後に、そのトランザクションは有効と見なされ、ブロックに追加されます。

3. 台帳の保管

各ピアノードは、ブロックチェーンの台帳（Ledger）のコピーを保持しています。これにより、ネットワーク内の複数のピアノードが台帳の状態を共有し、冗長性と信頼性を高めています。

4. チャネルに属する

Hyperledger Fabricでは、ネットワークを分割するためにチャネルという概念があり、チャネルごとに異なる台帳が存在します。ピアノードは特定のチャネルに属し、そのチャネルの台帳を管理し、そこで発生するトランザクションを処理します。

5. スマートコントラクトの実行

ピアノード上でチェーンコード（スマートコントラクト）をインストールして、トランザクションを処理するために使用します。これにより、ビジネスロジックをピアノードが実行し、トランザクションの内容に基づいて台帳を更新します。

6. 注文者ノードとの連携

ピアノードは、トランザクションを処理した後、**注文者ノード（Orderer Node）**にトランザクションを送信し、そこでコンセンサスアルゴリズムによってブロックが作成されます。注文者ノードがトランザクションをブロックにまとめ、再びピアノードにそのブロックを配信します。

Amazon Managed Blockchain上でも、これらのピアノードがHyperledger Fabricネットワーク内で機能し、分散型アプリケーションの動作を支えています。ピアノードをデプロイし、ネットワークに参加させることで、チェーンコードの実行や台帳の管理を行うことができます。
